# wk-code-review

Claude Code 代码审查 Skill — 通过对比当前分支与基准分支进行代码审查。

## 功能特性

- **多维度审计**：覆盖规范审计、业务逻辑、方案合理性、性能、可靠性（Istio）、安全审计 6 个维度
- **智能分批**：根据变更文件数量自动分组，最多 4 个 Agent 并行执行
- **中断恢复**：支持审计中断后从断点继续，不丢失进度
- **EliteForge 规范集成**：内置 Java、MyBatis、Spring、Git 等开发规范检查规则

## 安装

```bash
npx skills add whalezhao/wk-code-review -g -y
```

## 使用方法

在 Claude Code 中直接调用：

```
/wk-code-review <branch-name>
```

或描述触发：

```
帮我对 feature/xxx 分支进行代码审查
代码审计当前分支
```

## 审计维度

| 维度 | 说明 | 权重 |
|------|------|------|
| 规范审计 | 命名、日志、异常、事务、MyBatis、Spring 规范 | ×1.5 |
| 业务逻辑 | 需求覆盖、边界值、null 检查、数据一致性 | ×1.5 |
| 方案合理性 | N+1 查询、重复代码、过度设计 | ×1 |
| 性能 | 分页、批量操作、异步超时 | ×1 |
| 可靠性 | 幂等、状态外化、分布式锁、MQ 可靠性 | ×2 |
| 安全审计 | SQL 注入、XSS、敏感信息、权限校验 | ×3 |

## 审计结论判定

- 任何 CRITICAL × 安全/可靠性 → **阻塞合并**
- 任何 CRITICAL → **需整改**
- 加权 WARNING 分数 ≥ 3 → **需整改**
- 加权 WARNING 分数 < 3 → **通过**

## 输出

审计完成后生成 Markdown 报告，包含：

- 问题统计汇总（按严重程度 × 维度）
- 问题详细列表（按文件分组）
- 修复优先级建议
- 审计结论判定
